模擬規則
1. 每次掃描 Binance USDT-M 市場，計算 NOW_LONG / NOW_SHORT / PLAN_LONG / PLAN_SHORT。
2. 開啟模擬後，NOW 會用目前價格模擬市價開單；PLAN 只會建立條件單，到價才模擬成交。
3. 後續每次掃描或在模擬頁按「立即刷新數據」，都會用最新價格更新每張單的未實現 PnL；觸及停損或目標即自動平倉並轉成已實現 PnL。
4. 手續費以「單邊手續費 %」計入開倉與平倉。
5. 手機通知可用於模擬開倉、條件單成交、TP/SL 平倉；若系統通知被瀏覽器限制，仍會用頁面提示、聲音、震動提醒。
6. 這不是回測，這是前向紙上交易；要連續跑幾天，才看得出策略是否有正期望值。
7. 20EMA / 30MA / 200MA 距離會納入評分：顯示「高於/低於」與顏色標記；方向對但離均線太遠，會被視為追價風險而降分。
8. 交易紀錄建議定期匯出 JSON；JSON 可完整還原，CSV 適合做表格分析，交易摘要適合直接貼給我判讀策略。
9. 若自動掃描卡住，先按「停止 / 重置」，再按「自動掃描」；若仍失敗，看掃描頁下方錯誤訊息。
10. v6.8 新增動態出場：只在單子曾經浮盈後啟動保護，目標是保留原本 TP 能力，同時減少 DASH 這種「先賺後打完整 SL」的情況。
11. v7.0 把模擬頁選項收進右側側邊欄，並在模擬頁主畫面新增純圖示刷新按鈕。
12. v7.3 新增執行防呆：NOW 才能市價開單；PLAN 只用突破/跌破觸發價建立條件單；開單前保存訊號快照，避免 MIXED / 低量 / 未突破訊號被誤開。
